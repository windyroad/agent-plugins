P141 Phase 2: amends `itil-changeset-discipline.sh` to permit multi-commit slices targeting the same plugin to share one changeset. The Phase 1 hook (shipped 2026-05-02) denied any `packages/<plugin>/` commit without a staged `.changeset/*.md`, forcing AFK orchestrator iters that ship one logical bump across multiple ADR-014-grain commits to author N redundant changesets (e.g. P346 Phase 3 across 4 commits authored 2 `.changeset/p346-phase-3-*.md` files for one `@windyroad/itil` minor bump, rendering 2 CHANGELOG bullets for one release entry). Phase 2 adds Check 2b to `lib/changeset-detect.sh::detect_changeset_required`: an in-scope `.changeset/*.md` (or held-window `docs/changesets-holding/*.md` entry per ADR-042 Rule 7) targeting `"@windyroad/<plugin>": <any-bump>` via YAML frontmatter satisfies the gate for subsequent same-plugin commits. Scope = unpushed-range additions (`<base>..HEAD` with base = `@{u}` upstream tracking branch, fallback `origin/main`) + untracked working-tree files + modified-not-staged working-tree files. Once a changeset is consumed onto `origin/<base>` (drained by changesets-action at release time), it no longer counts and a fresh changeset is required for the next slice. Per-plugin granularity preserved: an `@windyroad/itil` changeset does NOT cover a `packages/voice-tone/` commit. Phase 1 strict-deny semantics preserved when neither Check 2a (staged) nor Check 2b (in-scope) is satisfied. SKILL guidance amended in `manage-problem` Step 11 to document the multi-commit slice pattern. Behavioural bats per ADR-005 + P081: 8 new fixtures (in-range positive, consumed-to-origin boundary, wrong-plugin negative, untracked positive, modified-not-staged positive, frontmatter-wrong-slug negative, held-window in-range positive, no-base fail-open-to-Phase-1). Full hooks/test/ suite green: 377/377.
