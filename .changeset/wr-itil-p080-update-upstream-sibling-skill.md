feat(itil): /wr-itil:update-upstream sibling skill — bidirectional lifecycle update for upstream-reported problems (P080 Phase 1)

Closes the P080 outbound-lifecycle-update leg of the reporter loop. When a local problem ticket transitions through its lifecycle (Open → Known Error → Verification Pending → Closed) and carries a `## Reported Upstream` section, the new `/wr-itil:update-upstream` sibling skill drafts a transition-specific lifecycle-update comment, composes it through the external-comms risk gate (`wr-risk-scorer:external-comms`) and voice-tone gate (`wr-voice-tone:external-comms`), auto-posts via `gh issue comment` when both gates pass within appetite, and queues an `outstanding_questions` entry when either gate fails above appetite (P352 queue-and-continue — does NOT halt the orchestrator). On Verifying → Closed the skill also runs `gh issue close` so the upstream tracker matches local `.closed.md` state. Reciprocal sibling to `/wr-itil:report-upstream` per ADR-010 amended Skill Granularity.

Trigger surfaces wired into `/wr-itil:transition-problem` Step 7b (new advisory subsection) AND `/wr-itil:manage-problem` in-skill Step 7 (copy-not-move per ADR-010 amended P093 for the in-skill callers: Step 9b auto-transition, Step 9d closure inside review, the Parked path). Fires unconditionally on every transition — the sibling skill's no-op exit absorbs the common case of tickets with no `## Reported Upstream` section.

ADR-024 amended (P080 entry at top of `## Amendments`) — bidirectional lifecycle-update contract, transition templates per status pair, gate composition (AND not OR — both gates must pass), `## Upstream Lifecycle Updates` append-only audit-trail back-write surface, `## Queued Upstream Update` queue surface (distinct from the initial-filing `## Queued Upstream Report` renamed per the 2026-06-04 second-amendment leaf (c) — distinct sections so one ticket can carry both without collision).

R009 prose-floor discharge per ADR-061 Rule 4 evidence-floor + ADR-075 Amendment 2026-06-02: paired promptfoo Tier-A/B eval at `packages/itil/skills/update-upstream/eval/promptfooconfig.yaml` ships in the same commit as the SKILL prose (mirrors the P324 Phase 6 / RFC-019 P355 pattern). 23 behavioural bats at `packages/itil/skills/update-upstream/test/update-upstream-contract.bats` all green. Phase 2 deferred: `--catchup` migration mode for retroactive coverage of existing `.verifying.md` / `.closed.md` tickets, and live end-to-end test against an upstream issue.

User direction ratified verbatim 2026-04-26 (recorded inline in the P080 ticket): (a) new sibling skill not a mode flag; (b) risk + voice-tone external-comms gated then auto-comment; (c) same composition pattern as inbound (P079).
