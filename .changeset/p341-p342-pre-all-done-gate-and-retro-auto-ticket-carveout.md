P341 + P342: AFK loop session-wrap gate sequence + iter-retro auto-ticket trust-boundary.

**P341 — work-problems Step 2.4 Pre-`ALL_DONE` gate sequence.** `packages/itil/skills/work-problems/SKILL.md` now carries a new `Step 2.4 — Pre-`ALL_DONE` gate sequence` step that fires UNCONDITIONALLY before every `ALL_DONE` emit (every stop-condition, every halt-path with a final summary, quota-exhaustion natural end). The sequence has three ordered gates: (a) outstanding-questions surface via Step 2.5b (read `.afk-run-state/outstanding-questions.jsonl`; surface via `AskUserQuestion`-when-available-else-table; truncate queue); (b) session-level retro via `/wr-retrospective:run-retro` (orchestrator-main-turn retro, distinct from per-iter retro per P086; retro commits its own work per ADR-014); (c) emit `ALL_DONE` ONLY after (a) AND (b) complete. Hard-fail mode: if either gate cannot complete cleanly, halt with directive instead of `ALL_DONE` — recoverable via user satisfying the gate and re-invoking. Output Format section + Non-Interactive Decision Making table now reflect the gate-sequence-driven `ALL_DONE` position. Closes the structural gap that allowed `ALL_DONE` to emit while direction-class observations remained queued AND without a session-level retro running. 2026-05-31 user direction verbatim drove the contract; the sibling `/wr-itil:work-problem` (singular) is pick-and-run with no `ALL_DONE` and is unaffected.

**P342 — iter-retro auto-ticket trust-boundary mirror.** `packages/itil/skills/work-problems/SKILL.md` Step 5 iter-prompt body now relaxes the "no `capture-*` siblings mid-loop" rule for RETRO-surfaced observations specifically and adds the P342 classification taxonomy: recurring class-of-behaviour / SKILL-contract drift / hook misbehaviour / framework-gap observations route to `/wr-itil:capture-problem` (mechanical-stage carve-out per `/wr-retrospective:run-retro` Step 4a precedent); direction-setting observations (design choice, deviation-approval, framework boundary) route to `outstanding_questions` for orchestrator-level Step 2.5 surface; ambiguous defaults to auto-ticket per the trust-boundary asymmetry that closes the silent-queue-accumulation hazard. `packages/retrospective/skills/run-retro/SKILL.md` Step 4b Stage 1 now carries the symmetric mirror — same taxonomy fires whether retro runs in iter context (work-problems Step 5) OR standalone in main turn (run-retro Step 4b). Per-observation `AskUserQuestion` would re-route mechanical decisions back to the user (lazy-deferral surface per Step 2d Ask Hygiene Pass) and is forbidden by the carve-out's framework-resolved authority (ADR-013 Rule 5 + ADR-044). The "no `capture-*` siblings mid-loop" rule is preserved for non-retro mid-iter capture (P078-class spam); the carve-out is bounded to the retro path.
