Decompose a Fix Into Coordinated Changes

STORY-MAP-002 · JTBD-008 · the developer's end-to-end journey turning a problem into a shipped, coordinated fix

When my fix to a problem spreads across multiple coordinated changes, I want the work shaped, time-boxed, and traced above the commit level — so each piece is a first-class entity, not buried in a ticket body.

End: the adopter who hit the problem gets a coordinated fix in a published release, with a durable trace from problem → RFC → change.
