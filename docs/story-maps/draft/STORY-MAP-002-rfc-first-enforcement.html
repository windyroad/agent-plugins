STORY-MAP-002: RFC-first enforcement at fix-time

Story map for the RFC-first rework (RFC-005). It decomposes the work that makes the only avenue to fix a problem the implementation of its RFC — the gate refuses fix-work without a pre-existing RFC, routes to authoring, and escalates a genuinely-new decision to a ratified ADR. Backbone × ribs × slices layout per Patton, User Story Mapping (O'Reilly, 2014).

Backbone (the maintainer's / AFK orchestrator's journey when a fix is proposed on a Known Error, left → right): Detect the RFC precondition → Propose the fix (interactive) → Propose the fix (AFK) → Author RFCs as story maps → Prove it behaviourally → Clear the legacy debt.

The ribs below are release slices (walking skeleton first); each slice links its INVEST story card. All slices are draft — authored ahead of the enforcement that will require them (dogfooding RFC-first: no enforcement needed to dogfood it).

Release 1 — the gate (walking skeleton: detect → propose in / out)

Detect — STORY-012: predicate refuses / routes to authoring (not auto-create) [was B12/B3] Propose (interactive) — STORY-013: manage-problem authors RFC first; uncovered choice → ratified ADR [was B13/B4] Propose (AFK) — STORY-014: work-problems authors first; lawfully halts on an uncovered decision [was B14/B5]

Release 2 — author + prove

Author RFCs — STORY-015: capture-rfc/manage-rfc author a pre-implementation story map; retire --fix-time [was B15/B8+B11] Prove — STORY-016: behavioural bats (RFC-less → refused; uncovered → blocks for ADR; covered → proceeds) [was B16/B6]

Release 3 — clear the legacy debt

Clear debt — STORY-017: backfill-or-supersede the skeleton RFCs the repudiated mechanism left behind [was B17/B11-backfill]
