# Codex CLI as second runtime

> Captured via /wr-architect:capture-adr (foreground-lightweight aside-invocation per ADR-032 P156 amendment). Run /wr-architect:create-adr on this ID to expand the deferred sections canonically.

## Context and Problem Statement

`@windyroad/*` plugins have shipped Claude Code only; ADR-002 (Monorepo per-plugin packages) line 241 and ADR-003 (Marketplace-only distribution) line 93 named Codex/Copilot support as a reassessment trigger but did not record the choice. OpenAI's Codex CLI (codex-cli 0.137.0) reached a stable plugin marketplace + custom-collaborator-agent surface, so the trigger fired.

## Considered Options

1. **Option A (chosen)** — Dual-runtime in shared `packages/shared/install-utils.mjs`: extend `parseStandardArgs` with `--runtime claude|codex|both` and the helper API with `addCodexMarketplace` / `installCodexPlugin` / `updateCodexMarketplace` / `uninstallCodexPlugin`; each plugin ships `.codex-plugin/plugin.json` mirroring `.claude-plugin/plugin.json` (sync'd by `scripts/sync-codex-plugin-manifests.mjs`); the plugin's existing `agents/agent.md` is the source of truth and Codex auto-discovers it as `<plugin>:<agent-md-name>` (e.g. `wr-architect:agent`); a repo-root `.codex/agents/<plugin>.toml` generated by `scripts/sync-codex-agents.mjs` is a source-repo dogfooding alias only (not shipped to adopters); repo-root `AGENTS.md` is the Codex sibling of `CLAUDE.md`, kept aligned with `docs/agent-instructions/shared-governance.md` (runtime-neutral) via `scripts/check-agent-instructions.mjs`. Rollout grain: architect-as-pilot — `wr-architect` is the proof, other 12 packages follow per-iter (their `lib/install-utils.mjs` already carries the API via sync, but their `bin/install.mjs` does not expose `--runtime` yet and they have no `.codex-plugin/` manifest).
2. (deferred — see /wr-architect:create-adr canonical review)

## Decision Outcome

Chosen option: **"Option A"**, because the shape matches ADR-002's explicit forward-looking note ("the per-plugin package structure should extend naturally — each package adds its tool-specific install logic") and ADR-017's sync-script + drift-check discipline. Verified end-to-end with codex-cli 0.137.0: `codex plugin marketplace add .` + `codex plugin add wr-architect@windyroad-local` + `codex exec --cd <adopter-dir>` spawning `wr-architect:agent` from a no-`.codex/` adopter directory returned a correctly-shaped `Architecture Review: ISSUES FOUND / [Undocumented Decision]` verdict.
