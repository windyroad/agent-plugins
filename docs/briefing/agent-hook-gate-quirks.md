## What Will Surprise You

- **PostToolUse `architect-mark-reviewed.sh` writes the gate marker ONLY on a fresh `Agent` tool call — NEVER on a `SendMessage` resume of an existing architect agent.** Two compounding quirks: (1) the hook parses the verdict by anchored grep on the `**Architecture Review: PASS**` / `**ISSUES FOUND**` heading — an `ISSUES FOUND` first verdict writes no marker (FAIL branch); (2) more importantly, the hook is a `PostToolUse:Agent` hook, so resuming the architect via `SendMessage` to upgrade ISSUES→PASS does NOT fire it at all — even a clean `**Architecture Review: PASS**` from the resumed agent leaves the marker unwritten. So the "re-invoke OR SendMessage asking for a PASS heading" workaround is HALF WRONG: SendMessage-resume will not set the marker. Recovery after a genuine approving PASS (verified 2026-06-28, P399 iter): either (a) fresh-spawn a new `wr-architect:agent` whose output leads with `**Architecture Review: PASS**` (fires the PostToolUse, writes marker+hash), or (b) the sanctioned manual assertion — `touch /tmp/architect-reviewed-$SID && rm -f /tmp/architect-reviewed-$SID.hash` (marker-present + no-hash → `architect-gate.sh` allows AND skips the docs/decisions drift re-check, immunising subsequent same-session edits; SID = newest `architect-plan-reviewed-*` / `architect-announced-*` basename). Use (b) when you already hold a genuine PASS and want to avoid a wasteful full re-review. <!-- signal-score: +2 | last-classified: 2026-06-28 | first-written: 2026-05-11 -->
- **`find /tmp -name PATTERN` silently returns zero on macOS without a trailing slash** — because `/tmp` is a symlink to `private/tmp` and `find` does not traverse the symlink unless the path ends with `/` (or `-L` is set). Bites shim helpers that enumerate session markers (`mark-oversight-confirmed.sh`, `manage-problem` create-gate enumeration) — they exit 0 with no markers written even when the announce markers exist clearly via `ls`. Verified 2026-06-17 in the ADR-082 oversight drain (P368 root cause): `find /tmp -name 'architect-announced-*'` returned empty but `ls /tmp/architect-announced-*` returned 93 files. Workaround at call site: `SESSION_MARKER_DIR=/tmp/ wr-architect-mark-oversight-confirmed <path>`. Real fix: helper invocations should use `find "${MARKER_DIR%/}/" -name ...` or `find -L "$MARKER_DIR" -name ...`. Audit grep: `grep -rn 'find /tmp ' packages/*/scripts/ packages/*/hooks/`. <!-- signal-score: +2 | last-classified: 2026-06-17 | first-written: 2026-06-17 -->
- **`grep PATTERN file.md` silently returns zero on markdown files containing em-dashes (or other Non-ISO extended-ASCII)** — `file docs/decisions/README.md` reports "Non-ISO extended-ASCII text" and grep classifies it as binary, suppressing matches. `grep -ac "PATTERN"` (force-text) returns the real count; bare `grep -c` returns 0 even when the substring exists clearly. Bites any retro / drift-check / compendium audit that greps these files. Verified 2026-06-17 during the ADR-082 compendium verification: `grep "ADR-082" docs/decisions/README.md` returned nothing but `grep -a "ADR-082"` returned the entry correctly. Always use `grep -a` (or `LC_ALL=C` plus `-a`) when grepping generated markdown that may carry em-dashes. <!-- signal-score: +2 | last-classified: 2026-06-17 | first-written: 2026-06-17 -->
- **Outstanding-questions queue accumulates across AFK loops without an automatic drain surface** — `.afk-run-state/outstanding-questions.jsonl` collects substance-confirm questions queued during AFK iter subprocesses (where the iter can't fire AskUserQuestion mid-loop). P341 shipped a Step 2.4 Pre-ALL_DONE gate intended to surface these on loop end, but the gate's actual drain scope was insufficient (cross-session accumulation observed 2026-06-17: 15 entries spanning iter-1 through iter-31 across multiple sessions, untouched until user pointed at the file). Workaround: invoke `/wr-architect:review-decisions` + `/wr-jtbd:confirm-jobs-and-personas` regularly as the cross-session drain surfaces; explicitly read `.afk-run-state/outstanding-questions.jsonl` at session start to triage stale entries. Real fix: P341 verification-failure recorded; structural mechanism deferred to next `/wr-itil:transition-problem` or `/wr-itil:review-problems` pass. <!-- signal-score: +2 | last-classified: 2026-06-17 | first-written: 2026-06-17 -->
