## What You Need to Know

- **Parallel Claude sessions can create P-ticket numbering collisions.** `manage-problem` / `create-adr` compute `max(local, origin)+1` automatically; `work-problems` fetches origin at loop start. Outside the skills, `git fetch origin` and check the highest P-number before opening tickets.
- **ADR-020 auto-release is skipped inside AFK loops** (ADR-018 Step 6.5, avoids double-release). A fix scoped to non-AFK governance paths is NOT exercised by `/wr-itil:work-problems` iterations — when evaluating Fix Released tickets for closure, check which code path actually fired.
- **Verify Fix Released tickets from your own in-session evidence** before deferring to the user. CI passes, preflight runs, release pipeline behaviour, next-ID assignments, and commit-gate scoring are all directly observable. Only leave `needs user verification` for genuinely user-side exercise (interactive skill invocations not triggered in session, plugin-cache-lag, behaviours visible only in other projects). At AFK loop stop, proactively surface close-candidates whose fix paths were exercised this session. (verifying P048 + P053)
- **At AFK loop stop, surface outstanding design questions before emitting `ALL_DONE`.** Skipped tickets whose blockers are user-answerable (naming, direction, pacing, scope) belong in a batched `AskUserQuestion` (cap 4). Don't treat "needs interactive input" as terminal. (verifying P053)
- **Every AFK halt path that emits a final summary must route prior-iter accumulated user-answerable skips through Step 2.5b first** (Step 0 session-continuity / fetch-failure, Step 6.5 CI-failure / above-appetite, Step 6.75 dirty-unknown). Step 2.5b is the single source of truth for the AskUserQuestion-when-available-else-table logic; it surfaces *prior-iter accumulated skips only* — it does NOT ask the user how to remediate the halt cause itself (CI failure / above-appetite / dirty-unknown remain halt-with-bug-signal). (verifying P122 + P126)
- **Any gate-class hook released mid-AFK-loop is ineffective for the immediate-next iter** — the iter subprocess loads the pre-release cached plugin version. Compounds with `claude plugin install` no-op-when-already-installed. Workaround: invoke `/install-updates` between `release:watch` completion and next-iter dispatch. ## What Will Surprise You

- **Orchestrator main-turn mid-loop `AskUserQuestion` between iters is forbidden per ADR-044, even on consecutive-iter target picking.** SKILL contracts prescribing WSJF + ticket-state + tie-break ladder for next-iter selection have framework-resolved the decision; asking the user sub-contracts framework-mediated work back. A Phase 2b structural-enforcement hook (UserPromptSubmit / Stop detecting AskUserQuestion in mechanical-stage zones) is the load-bearing fix beyond the per-skill SKILL-surface refactor. (verifying P130 / P132) <!-- signal-score: +2 | last-classified: 2026-05-25 | first-written: 2026-05-17 -->
- **Iter API stream-timeout class — work is salvageable from the working tree if bats green.** When an iter subprocess returns `is_error: true` with `API Error: Stream idle timeout` AFTER staging coherent work but BEFORE `git commit`, the staged work is in the working tree. Recovery: orchestrator runs the iter's bats fixtures; if green, commits the staged work from the main turn with attribution. Distinct from the SIGTERM exit-flush class — stream-timeout preserves metadata in the JSON envelope AND staged files. (verifying P246; P261 tracks the Step 5 exit-code contract amendment) <!-- signal-score: +2 | last-classified: 2026-05-25 | first-written: 2026-05-18 -->
- **Subprocess dispatch via Bash `run_in_background=true` with `&` kills the subprocess on parent exit** — the backgrounded shell sends HUP to any child that hasn't detached, producing a 0-byte JSON file. Working pattern: put the ENTIRE dispatch+poll loop (the `claude -p` invocation + `while kill -0 $ITER_PID` poll + `wait $ITER_PID` + cost-summary echo) into ONE shell script and run THAT via `run_in_background=true`. The script lives long enough to keep the subprocess attached. <!-- signal-score: +2 | last-classified: 2026-05-25 | first-written: 2026-05-18 -->
