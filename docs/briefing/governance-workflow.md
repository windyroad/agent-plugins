# Governance Workflow

Cross-session learnings about ADRs, architect and JTBD reviews, risk scoring, voice-tone, and the session-wide unifying patterns (progressive disclosure, canonical+sync, SKILL+REFERENCE).

## What You Need to Know

- **Risk appetite is Low (4)**. Changes scoring Medium (5+) need explicit acknowledgement. See `RISK-POLICY.md`.
- **All ADRs in `docs/decisions/` are still `.proposed.md`** — none ratified. Amendments are cheap: prefer amending over superseding. When a P-problem intersects a proposed ADR, revise the ADR rather than adding a compatibility clause.
- **ADR-002 plugin dependency graph lists `@windyroad/tdd` as standalone.** Cross-plugin features (e.g. P018 JTBD traceability, P017 shared concern-splitting helper) must update the graph explicitly; don't silently add deps.
- **Same-day ADRs may conflict — cross-reference before implementing a ticket direction.** A direction decision added to a ticket today and a new ADR accepted the same day on the same files may carry different execution models (e.g. synchronous Step-0 delegation vs asynchronous background-subagent invocation). The architect catches this, but the wasted iteration can be avoided by greping `docs/decisions/*.proposed.md` for files dated the same day as the direction and spot-checking the overlap before writing implementation code. Observed 2026-04-20 AFK iter 2: P014's 2026-04-20 direction and ADR-027's 2026-04-20 acceptance collided on manage-problem/run-retro/create-adr SKILL.md files; architect flagged the conflict and the iteration skipped with findings. See P014.
- **Architect reviews routinely surface 1-3 "bundle these in the same commit" follow-up edits.** A commit that touches ADR-X may require a one-line edit to ADR-Y (explicit cross-reference), or an adjacent skill's SKILL.md carve-out, or a missing rationale section. Factor this into effort estimates — "L" is usually "L + architect-added bundling" at the upper bound, not the midpoint. Observed repeatedly 2026-04-20 AFK iters 3/4/5. This is the architect working as designed; don't resist it — bundle.
- **Progressive disclosure is the session-wide unifying solution pattern for context-budget problems** per ADR-038 (2026-04-22). Less info upfront + explicit affordances (agent pointers, REFERENCE.md paths, turn-1 references) so consumers expand on demand rather than having context re-injected. Companion axes: trimming, consolidation, gating, scope-awareness. Applied first to UserPromptSubmit hooks (P095 shipped); candidates at PreToolUse/PostToolUse (P096), SKILL.md runtime size (P097), user/project-owned files (P098). Anchor ticket P091.
- **Canonical+sync is the pattern for cross-plugin shared code** (ADR-017 for `.mjs`, ADR-028/ADR-038 extend to `hooks/` and `hooks/lib/`). Three-step scaffold: (1) canonical at `packages/shared/{,hooks/lib/}<name>`, (2) per-plugin synced copies, (3) `scripts/sync-<name>.sh` with `--check` mode + npm `sync:<name>` / `check:<name>` scripts + CI drift step + `packages/shared/test/sync-<name>.bats`. Each plugin sources its LOCAL copy, never the canonical — preserves ADR-002 installable-independently. Precedents: install-utils.mjs (P026), session-marker.sh (P095).
- **SKILL.md + sibling REFERENCE.md is the progressive-disclosure pattern for skill bodies** (ADR-038 applied to skill runtime size — formal ADR expected on P097 when the pattern is generalised across the windyroad suite). Shape: `SKILL.md` carries runtime steps, bash snippets, report templates, contract invariants, and explicit pointers to `REFERENCE.md` anchors where deeper rationale is needed; `REFERENCE.md` carries BRIEFING references, per-ADR rationale, edge cases, scope exclusions, and the "not in scope" list. Consumer loads REFERENCE on demand, not eagerly. Reference implementation: `.claude/skills/install-updates/{SKILL,REFERENCE}.md` (P098, 2026-04-22). Before generalising across windyroad plugins, P097 should author a sibling ADR naming a byte budget per SKILL.md (discoverable via `performance-budget-*` glob per ADR-023).

## What Will Surprise You

- **Anthropic's official `skill-creator` skill ships a mature eval harness** (evals.json + dual-run with-skill/baseline + grader subagent + aggregate_benchmark + HTML viewer). Directly applicable to testing our SKILL.md documents. See P012 for how it maps to the skill-testing-harness decision.
- **The JTBD reviewer agent sometimes returns a bare verdict ("FAIL" or just a file list) without remediation reasons.** Re-query with explicit framing ("explain specifically which job conflicts and what would need to change") to get actionable guidance. See P037.
- **Skill-creator eval subagents can hit rate limits mid-batch.** Spawning 6 parallel subagents (3 with-skill + 3 baselines) risks 2-3 failing with `"You've hit your limit"`. Run fewer in parallel, or accept partial results and grade what lands.
- **"Wrong Approach" (54) and "Buggy Code" (41) dominate friction types over 30 days of sessions** per the Claude Code insights report. Root cause: prompts that lump diagnose + implement + release together let autonomous loops race past diagnosis into rework. Violates JTBD-006's scope-expansion outcome and JTBD-201's evidence-before-mitigation principle. See P039.
- **External-facing text (GitHub comments, npm READMEs, issue replies, RapidAPI listings) has no voice-and-tone gate.** AI-sounding output — em-dashes, hedging, posting "keep open" on 2-year-old issues — triggers recurring "FFS" corrections. The `wr-voice-tone` plugin governs in-repo text only. See P038.
- **`git ls-tree <ref> <path>` without `--name-only` returns lines containing blob SHAs**, which contain digit substrings that false-match `<NNN>` regex patterns. The skill-provided next-ID bash in `manage-problem` Step 3 and `create-adr` Step 3 currently has this bug — a `grep -oE '[0-9]{3}'` over raw ls-tree output sometimes picks 3-digit runs from a blob SHA. Use `git ls-tree --name-only <ref> <path>` + `sed 's|^<path>/||'` + `grep -oE '^[0-9]+'`. Observed 2026-04-20 opening P055: local_max=054 correct but origin_max=997 wrong (from a blob hash). Caught via visual sanity-check.
- **Verify JTBD IDs against `docs/jtbd/<persona>/` before citing them in an agent prompt — don't cite from memory.** Job IDs look like `JTBD-NNN` and are cheap to misremember. When you get them wrong the jtbd-lead will (correctly) flag the misattribution and the review round has to be re-run. Observed 2026-04-20 when drafting ADR-030's rationale: I cited "JTBD-005 (keep multi-project workflow coherent)" from memory; actual JTBD-005 is "Invoke Governance Assessments On Demand". Cost: one round of JTBD review. Fix: quickly `ls docs/jtbd/<persona>/` + read the job title's first few lines before citing.
- **Send the full draft — not a summary — to wr-architect:agent for ADR or design review.** The architect can't flag what it doesn't see. A condensed prompt that omits MADR sections will produce "missing sections" feedback even when the draft has them all. Observed 2026-04-20 reviewing ADR-030: my condensed prompt omitted Decision Drivers, Considered Options, and Good/Neutral consequences; architect flagged all three as missing even though the draft already included them. Cost: one wasted review round. Fix: quote the full draft inline, even if long — the architect tool call accepts it.
- **A "Read first each session" directive in a project `CLAUDE.md` creates a new eager-load path — it does NOT surface a pre-existing affordance.** Before writing a pointer like `docs/X.md — read first each session`, check whether X.md was already discoverable at session start (via global `~/CLAUDE.md`, a UserPromptSubmit hook, or similar). If not, the directive adds a session-start context cost that didn't exist before. Prefer letting the owning plugin advertise via its own UserPromptSubmit hook (the pattern 6 of 8 windyroad plugins already follow — P100 tracks adding it to `wr-retrospective`). Observed 2026-04-22 P098 amendment: the P098 fix initially hand-authored pointers for `docs/BRIEFING.md` + redundant pointers to `docs/decisions/` / `docs/jtbd/` / `RISK-POLICY.md` / `docs/STYLE-GUIDE.md` / `docs/VOICE-AND-TONE.md`; user corrected: "we shouldn't have to add these things to CLAUDE.md. It should be automatic." 5 of 7 pointers were redundant with existing plugin hooks; 1 was a new regression; 1 was contextual (correctly on-demand).
