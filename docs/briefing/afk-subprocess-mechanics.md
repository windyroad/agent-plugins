# AFK Subprocess Mechanics

`claude -p` subprocess dispatch shape, output-format JSON contract, cost metadata extraction, and the smaller subprocess-vs-orchestrator quirks. Sibling to `afk-subprocess.md` (overview + critical surface) and `afk-subprocess-recovery.md` (SIGTERM, iteration-worker discipline, mid-loop asking). Split out 2026-05-03 per P145 MUST_SPLIT contract.

## What Will Surprise You

- **Subagents spawned via the Agent tool CANNOT spawn other subagents — a hard platform restriction.** `Agent(agent_type)` has no effect in subagent definitions; direct invocation returns "Agent is not available inside subagents." Custom `subagent_type` definitions cannot lift this. Consequence: Agent-tool-spawned iteration workers cannot satisfy architect/JTBD PreToolUse edit-gate markers (only set by the Agent-tool PostToolUse hook) nor the risk-scorer gate, so iterations on gate-covered paths silently halt. (closed P084)

- **`claude -p` subprocess dispatch is the AFK iteration-isolation mechanism**, not Agent-tool spawning. The subprocess is a full main Claude Code session with Agent in its own surface: architect/JTBD/risk-scorer reviews run at full depth, PostToolUse markers fire matching the subprocess's own `$CLAUDE_SESSION_ID`, committed changes land natively. Dispatch flags: `claude -p --permission-mode bypassPermissions --output-format json <iteration-prompt>`. No `--max-budget-usd` cap (quota is the natural stop). Prompt cache reuse observed across subprocess invocations in the same Bash session. (ADR-032 subprocess-boundary sub-pattern; closed P084)

- **`claude -p --output-format json` response carries per-iteration cost/token metadata alongside `.result`.** Extractable fields: `.total_cost_usd`, `.duration_ms`, `.usage.{input,output,cache_creation,cache_read}_tokens`. **PII guard**: the JSON also carries `session_id`, `model`, `stop_reason`, `permission_denials`, `uuid` — do NOT extract or log these; they are subprocess-envelope fields.

- **`claude -p` invoked from Bash with `2>&1` needs `< /dev/null` to avoid a stdin warning contaminating stdout** — without it the "no stdin data received in 3s" warning is prepended to the JSON and breaks `jq`/`json.load` parsers. The dispatch shape in work-problems SKILL.md Step 5 carries the redirect. (closed P089)

- **`claude -p` cost-metadata authority hierarchy: `total_cost_usd` is cumulative; `.usage.*` may be final-turn-only.** When the subprocess exits via a background-task-notification ack, the JSON reports `num_turns: 1` with a small `duration_ms` and tiny `.usage.*` tokens — but `total_cost_usd` reflects the full subprocess lifetime. Trust `total_cost_usd` for dollar cost; treat `.usage.*` as best-effort. (closed P089)

- **Each `claude -p` subprocess writes its own `.jsonl` to the parent project log directory** (`~/.claude/projects/-<project-path>/<session_id>.jsonl`) with 24-90 tool_uses each. Cross-iteration full-trace retro via .jsonl parsing is feasible.

- **`claude -p` subprocess `SESSION_ID` is not exported in the shell environment** — session-scoped hook directories (e.g. the risk-scorer's `${TMPDIR}/claude-risk-${SESSION_ID}/`) resolve to a literal empty suffix, so manual marker creation from a Bash shell does NOT satisfy the gate (the hook's tool-input JSON carries a non-empty session ID pointing elsewhere). Implication: AFK iteration workers must either trigger the scorer subagent so the PostToolUse hook fires natively, or commit from the parent session. Resolved by the ADR-050 runtime-SID instrumentation hook (runtime marker fires on every Bash|Write|Edit|Read carrying the actual session_id from hook stdin JSON). (closed P108; verifying P142)

- **Subprocess iterations may intermittently block `Edit`/`Write` with a bare "haven't granted it yet" permission error** (distinct from hook `BLOCKED:` denies) even with `defaultMode: "dontAsk"` and fresh markers; the Bash surface is unaffected. On bare denied: retry once after re-invoking the relevant architect/JTBD agent to refresh the marker, else fall back to `python3 -c '...write_text...'` via Bash. (closed P092)

- **`Monitor` tool grep filter must match only terminal states, not any status line.** `gh run watch` reprints the full running-status view every 3s; a loose `grep -E 'completed|success|failed'` matches the same text on every refresh and fires repeatedly. Tighten with terminal-only patterns like `grep -E 'in [0-9]+s|conclusion=|Release complete\.|exit code'`.

- **Cached SKILL.md vs newly-installed hook can mismatch within a session.** Post-install + pre-restart, hooks update from cache while SKILL.md context does not reload. If a new hook denies on a contract the SKILL.md flow doesn't know about, set the required marker manually rather than re-running `/install-updates`. The hook source under `~/.claude/plugins/cache/windyroad/<plugin>/<version>/hooks/` is the truth of what the gate enforces. Same principle as "Plugin hooks run from marketplace cache, not source," applied across the SKILL.md/hook version boundary. (closed P119 / P120)
