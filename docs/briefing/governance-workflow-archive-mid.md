## What You Need to Know (archived 2026-05-13)

- **User direction: reject BUFD ranked-iteration of scorer suggestions** — the agent reads suggestions and decides what to do incrementally, like any other workflow decision. No upfront ranking, `action_class` columns, or pre-built executors. *"It should just get the suggestions and decide what to do, incrementally like we do for everything else."* (ADR-042, 2026-04-23). <!-- signal-score: 0 | last-classified: 2026-05-25 | first-written: 2026-04-23 -->
- **Gate approvals validate the proposed change, not the underlying hypothesis** — architect/JTBD/risk subagents only prove "this edit doesn't violate recorded rules"; they can't validate the edit's cause-effect link. Before committing to a hypothesis whose verification needs release+reinstall+restart+re-test, run the cheap diagnostic first (`claude plugin list`, `git log @{push}..HEAD`). The investigation-ordering analogue of JTBD-201 evidence-before-mitigation. (P113, 2026-04-24). <!-- signal-score: 0 | last-classified: 2026-05-25 | first-written: 2026-04-24 -->
- **Auto-create-on-missing-doc is the contract for doc-bearing review gates** — when `docs/VOICE-AND-TONE.md` / `docs/STYLE-GUIDE.md` / `docs/jtbd/` is missing, the `*-enforce-edit.sh` PreToolUse hook blocks the edit and routes to `/wr-<plugin>:update-guide` rather than silently disabling. New doc-bearing gates should follow this — block + route, never degrade to "review skipped". (2026-04-25). <!-- signal-score: 0 | last-classified: 2026-05-25 | first-written: 2026-04-25 -->
- **Post-landing ADR cleanup is a recurring shape** — when an ADR lands in 2-4 commits walking back its first draft, Rules get revised cleanly but Decision Drivers / Outcome / Consequences carry stale references. Budget a follow-on cleanup; reread any user direction dated AFTER the initial landing commit before amending, since framing may have evolved. An acknowledged JTBD Gap framed as "follow-up" may actually be deliberate policy. (ADR-042, 2026-04-24). <!-- signal-score: 0 | last-classified: 2026-05-25 | first-written: 2026-04-24 -->
