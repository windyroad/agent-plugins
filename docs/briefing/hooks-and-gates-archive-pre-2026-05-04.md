## What You Need to Know

- **PreToolUse:Write gate matchers on governance paths should be suffix-agnostic** — match on path-prefix (`docs/<path>/`) + content-shape (3-digit numeric basename `[0-9][0-9][0-9]-*`) rather than the trailing `.<status>.md` suffix, so ADR-031's per-state subdirectory layout survives without code change. `manage-problem-enforce-create.sh` (P119) is the reference impl. Apply to any future `docs/decisions/`, `docs/incidents/`, `docs/risks/` gate. <!-- signal-score: +1 | last-classified: 2026-05-25 | first-written: 2026-04-25 -->
- **`secret-leak-gate.sh` denies bats fixtures containing literal credential-shaped strings** — write the credential-shaped string at runtime via shell concatenation (`"g""hp""_AAAA..."`) or split a regex literal across concatenation boundaries (`'A''KIA[0-9A-Z]{16}'`) so the fixture file doesn't carry the literal pattern. The gate is correctly enforcing policy; this keeps test authoring possible without bypassing it. <!-- signal-score: +1 | last-classified: 2026-05-25 | first-written: 2026-04-26 -->

## What Will Surprise You

- **Each edit consumes the architect/WIP marker** — you need a fresh agent review for every blocked edit, not one per session. The risk-scorer / architect / JTBD / commit-gate bypass markers all carry a ~3600s TTL; long sessions can hit expiry between reviews and pay a re-review cycle. <!-- signal-score: +1 | last-classified: 2026-05-25 | first-written: 2026-04-19 -->
- **The risk-scorer PostToolUse hook matches agent names with regex dot (`.`), not literal characters** — a deliberate fix; don't "correct" it back to dashes. <!-- signal-score: +1 | last-classified: 2026-05-25 | first-written: 2026-04-19 -->
- **Edit gates block files outside the project** (e.g. `~/.claude/channels/discord/access.json`) — use bash to write non-project config files when gates are active. <!-- signal-score: +1 | last-classified: 2026-05-25 | first-written: 2026-04-19 -->
- **UserPromptSubmit hooks emit full MANDATORY prose only on the first prompt of a session** (ADR-038); subsequent prompts emit a ≤150-byte terse reminder (signal word + gate name + trigger artifact + `wr-<plugin>:agent` affordance). `wr-tdd` still emits dynamic state per-prompt. Empty `SESSION_ID` falls back to full block. Sessions started before the fix keep seeing full prose. <!-- signal-score: +1 | last-classified: 2026-05-25 | first-written: 2026-04-22 -->
- **The shared `lib/session-marker.sh` once-per-session helper is event-type-agnostic** — it gates by `${SYSTEM}` namespace + `${SESSION_ID}`, not by hook event, so it works for any always-on advisory body whose payload doesn't change within-session (incl. PreToolUse, e.g. `risk-scorer/plan-risk-guidance.sh`). Pick a unique system name, source the local copy, gate the body behind `has_announced` / `mark_announced`. <!-- signal-score: +1 | last-classified: 2026-05-25 | first-written: 2026-04-26 -->
- **`PostToolUse` hooks do NOT fire on pure assistant text turns** — they fire after tool invocations. Assistant-output validators must bind to `UserPromptSubmit` (preventive) and/or `Stop` (post-hoc: read the last turn from `transcript_path` on stdin, emit a `stopReason` nudge biasing the next turn — can't rewrite the emitted turn). `transcript_path` JSONL: each line `{type, message:{content}}`; filter `type==assistant`, concat `text` blocks. <!-- signal-score: +1 | last-classified: 2026-05-25 | first-written: 2026-04-24 -->
- **`SessionStart` is the correct surface for boot-time artefact announcement** (not UserPromptSubmit + marker). Matcher is a **string** on the **outer** entry object; values `startup`/`resume`/`clear`/`compact` (omit = all); only `type: "command"`; fires once per session on startup. First use: `packages/retrospective/hooks/session-start-briefing.sh` (ADR-040). <!-- signal-score: +1 | last-classified: 2026-05-25 | first-written: 2026-04-22 -->
