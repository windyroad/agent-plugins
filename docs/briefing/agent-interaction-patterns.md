- **A multi-question `AskUserQuestion` call can return answers for FEWER than all questions** â€” observed twice in one session (a 4-question batch returned 3 answers each time; the 4th question's answer was silently absent from the response). The dropped question is not flagged as unanswered â€” it simply doesn't appear in the returned answers map. When you batch N questions, check the returned answers against the questions you asked and **re-ask any that didn't come back** (a single follow-up `AskUserQuestion` with just the missing one). This is a Claude Code harness behaviour, not a windyroad-plugin defect â€” no plugin-side fix; the discipline is "verify answer-count == question-count, re-ask the gap." (2026-05-26; ADR-018 dropped from one batch, ADR-043 from another.)

- **On a genuine design fork, lead with your reasoning + a recommended option â€” don't open with a bare "how do you want to proceed?".** 2026-05-26 RFC pivot: the open form drew *"I want your opinion â€” what do you think?"*. Present the trade-off + a recommendation (architect-first for architecture forks), THEN `AskUserQuestion` to ratify. Distinct from the act-on-obvious rule above: that's for policy/direction-obvious calls (just act); this is for multi-path forks where the user wants your judgment surfaced first (the ratify ask still fires, just after your recommendation).
