## What Will Surprise You

> Older entries (3 archived 2026-05-31 + 1 × P302 decision-confirmation-prompts substance-first archived 2026-06-08) live in `agent-interaction-patterns-archive.md` for Tier 3 budget rotation. Load alongside this file for full history.

- **A multi-question `AskUserQuestion` call can return answers for FEWER than all questions** — observed twice in one session (a 4-question batch returned 3 answers each time; the 4th question's answer was silently absent from the response). The dropped question is not flagged as unanswered — it simply doesn't appear in the returned answers map. When you batch N questions, check the returned answers against the questions you asked and **re-ask any that didn't come back** (a single follow-up `AskUserQuestion` with just the missing one). This is a Claude Code harness behaviour, not a windyroad-plugin defect — no plugin-side fix; the discipline is "verify answer-count == question-count, re-ask the gap." (2026-05-26; ADR-018 dropped from one batch, ADR-043 from another.)

- **On a genuine design fork, lead with your reasoning + a recommended option — don't open with a bare "how do you want to proceed?".** 2026-05-26 RFC pivot: the open form drew *"I want your opinion — what do you think?"*. Present the trade-off + a recommendation (architect-first for architecture forks), THEN `AskUserQuestion` to ratify. Distinct from the act-on-obvious rule above: that's for policy/direction-obvious calls (just act); this is for multi-path forks where the user wants your judgment surfaced first (the ratify ask still fires, just after your recommendation).

- **Substance-confirm `AskUserQuestion` requires prose briefing + option-shaped + no-IDs-as-explainers + informed-without-external-lookup (P339 / P340).** Surfaced 2026-05-31 by ADR-078 commit `5196e3d`: a bundled Step 5 "review pass" `AskUserQuestion` ("does the problem statement + Decision Outcome (Option X) capture the situation? — yes/no/edits/different-option") landed `human-oversight: confirmed` on substance the user never explicitly ratified. The user had confirmed only draft quality; Option X was substantially different from the option the user actually wanted (Option 9). User correction: *"I never approved the scripted extraction. You are supposed to run decisions by me"* + *"How did that ADR skip ratification?"*. Five load-bearing requirements when surfacing substance for confirmation: (1) briefing in main-turn PROSE before the question fires (long question text is unreadable on mobile / accessibility tooling / notification surfaces; long prose + short question IS readable across the device matrix); (2) `AskUserQuestion` is option-shaped not yes/no (each considered option a selectable entry; user picks positively); (3) no IDs as explainers (the user does not have `ADR-NNN` / `P-NNN` / `JTBD-NNN` / `RFC-NNN` lookups on all devices — every option's substance self-contained in prose + label); (4) user can make an informed decision without external document lookup (briefing + question + options is self-contained); (5) substance-only at this fire (draft-quality, problem-statement-OK, consulted/informed belong to a SEPARATE later fire). Born-confirmed marker writes ONLY when the substance-confirm answer matches the draft on disk; mismatch triggers re-draft + re-fire (NOT a soft warn-and-proceed).
