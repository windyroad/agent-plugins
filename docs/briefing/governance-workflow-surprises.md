# Governance Workflow — What Will Surprise You

Cross-session "what will surprise you" learnings about ADR mechanics, JTBD reviewer behaviour, the `git ls-tree` blob-SHA next-ID trap, README-refresh reconciliation, and the smaller workflow gotchas that cost a wasted round when missed. Sibling to `governance-workflow.md`. Split out 2026-05-03 per P145 MUST_SPLIT contract for Tier 3 budget compliance.

## What Will Surprise You

- **Anthropic's official `skill-creator` skill ships a mature eval harness** (evals.json + dual-run with-skill/baseline + grader subagent + aggregate_benchmark + HTML viewer) — directly applicable to testing our SKILL.md documents. See P012. <!-- signal-score: 2 | last-classified: 2026-05-25 | first-written: 2026-05-03 -->
- **The JTBD reviewer agent sometimes returns a bare verdict ("FAIL" or just a file list) without remediation reasons.** Re-query with explicit framing ("explain specifically which job conflicts and what would need to change") to get actionable guidance. See P037. <!-- signal-score: 2 | last-classified: 2026-05-25 | first-written: 2026-05-03 -->
- **Skill-creator eval subagents can hit rate limits mid-batch.** Spawning 6 parallel subagents risks 2-3 failing with "You've hit your limit". Run fewer in parallel, or grade what lands. <!-- signal-score: 1 | last-classified: 2026-05-25 | first-written: 2026-05-03 -->
- **`git ls-tree <ref> <path>` without `--name-only` returns lines containing blob SHAs**, whose digit substrings false-match `<NNN>` next-ID regexes (a `grep -oE '[0-9]{3}'` over raw ls-tree output can pick a 3-digit run from a blob hash). Use `git ls-tree --name-only <ref> <path>` + `sed 's|^<path>/||'` + `grep -oE '^[0-9]+'`. <!-- signal-score: 2 | last-classified: 2026-05-25 | first-written: 2026-05-03 -->
- **Verify JTBD IDs against `docs/jtbd/<persona>/` before citing them in an agent prompt — don't cite from memory.** Misremembered `JTBD-NNN` IDs make the jtbd-lead flag the misattribution and force a re-run. `ls docs/jtbd/<persona>/` + read the title first. <!-- signal-score: 2 | last-classified: 2026-05-25 | first-written: 2026-05-03 -->
- **Send the full draft — not a summary — to wr-architect:agent for ADR or design review.** A condensed prompt that omits MADR sections produces "missing sections" feedback even when the draft has them all. Quote the full draft inline, even if long. <!-- signal-score: 2 | last-classified: 2026-05-25 | first-written: 2026-05-03 -->
- **A "Read first each session" directive in a project `CLAUDE.md` creates a new eager-load path — it does NOT surface a pre-existing affordance.** Before adding a pointer, check whether the doc was already discoverable at session start (global `~/CLAUDE.md`, a UserPromptSubmit hook, etc.). Prefer letting the owning plugin advertise via its own hook. <!-- signal-score: 2 | last-classified: 2026-05-25 | first-written: 2026-05-03 -->
- **JTBD reviewer returning FAIL on a partial-progress slice is often the expected case, not a blocker.** When a slice defers the writer-side half of a read/write pair, the reviewer correctly flags the persona's job breaking between slices. Resolve by (a) expanding the slice to include the writer side, (b) making the transitional artefact append-compatible, or (c) contracting the transition window. Pre-check producer/consumer pairs and bundle both halves unless the transitional shape absorbs the half-landed state. <!-- signal-score: 2 | last-classified: 2026-05-25 | first-written: 2026-05-03 -->
- **`grep -F 'pattern' file` parses a pattern beginning with `--` as a flag, not a literal** — `grep -F '--search' file` fails with `unrecognized option` (exit 2). Use `grep -F -- '--search' file`. In `run grep ...` bats blocks the failure is silent (reports `not ok` exit 2, looking like a missing-pattern miss). <!-- signal-score: 2 | last-classified: 2026-05-25 | first-written: 2026-05-03 -->
- **Per-operation README-refresh contracts (P094 + P062) cannot retroactively detect cross-session drift** — a past session that committed a ticket change while skipping the README staging step leaves a stale README no per-operation layer sees. The robustness layer is reconciliation as preflight: `packages/itil/scripts/reconcile-readme.sh` (diagnose-only: exit 0 clean / 1 drift / 2 parse error), wrapped by `/wr-itil:reconcile-readme` with agent-applied edits that preserve narrative content. `manage-problem` Step 0 + `work-problems` Step 0 invoke it before anything else; `transition-problem` deliberately does NOT (P062 already covers transition-time refresh). ADR-014 amended with the "Reconciliation as preflight" sub-rule. <!-- signal-score: 2 | last-classified: 2026-05-25 | first-written: 2026-05-03 -->
