## Rotated 2026-05-26 (Tier-3 split-by-date — oldest/settled entries from releases-and-ci.md)

- **GITHUB_TOKEN pushes don't trigger pull_request events**. The release preview uses `workflow_run` trigger instead. If adding new PR-triggered workflows for changesets PRs, use the same pattern.
- **npm won't overwrite a published version**. Preview publishes use pre-release suffixes (e.g., `0.1.2-preview.13`) to avoid blocking `changeset publish` from publishing the clean version to `latest`.
- **Multi-day commit accumulation surfaces latent test-fixture regressions on first push** — the regression often traces to an earlier commit in the chain that wiped a fixture source without a matching test sweep. Before wiping/renaming files referenced by tests, sweep `grep -r '<deleted-name>' packages/*/{scripts,skills,hooks}/**/test` first; synthesize fixture-local equivalents inline so tests aren't coupled to canonical state. ## Rotated 2026-06-08 (Tier-3 split-by-date — 2026-05-26 entries from releases-and-ci.md)

- **`push:watch` can false-green: it exits 0 at the push step before GitHub registers a CI run for the pushed tip sha.** It watches "every CI run anchored on HEAD"; if none is registered yet when it checks, it finds nothing failing and exits 0 — without ever watching a run. Observed 2026-05-26: push-watch exited 0, but `gh run list --commit <tip>` was empty for ~3 min, then the CI run appeared and ran 5 min. Do NOT treat push:watch exit-0 as "CI green" — confirm by polling `gh run list --commit <tip>` until a CI run registers, then `gh run watch <id> --exit-status`. Especially before a release. (Sibling of the P054/P060 stale-SHA class.) <!-- signal-score: 0 | last-classified: 2026-05-26 | first-written: 2026-05-26 -->

- **Authoring a `.changeset/*.md` triggers the external-comms review gate once per installed evaluator** (risk-scorer + voice-tone both gate `.changeset/*.md` writes per P073). Each denies the Write until ITS own marker exists. In-flight recovery: delegate to BOTH `wr-risk-scorer:external-comms` AND `wr-voice-tone:external-comms`, each with a prompt starting `SURFACE: changeset-author` and wrapping the changeset body — WITHOUT the `---` frontmatter, which the gate strips before hashing (P010) — in `<draft>...</draft>`. The PostToolUse mark hooks derive the keys; the retry Write of the identical body then passes. `BYPASS_RISK_GATE=1` is pre-session env only (P173). <!-- signal-score: 2 | last-classified: 2026-05-26 | first-written: 2026-05-26 -->

- **On patchy network, `push:watch`/`release:watch` fail at the long CI/publish waits; retry catches a window.** The bare `git push` lands even when the watch fails — the commit is pushed once `git rev-list origin/main..HEAD`=0, regardless of the watch. A git-only DNS failure while `npm`/`curl` work = stale resolver (post-Pty-restart), clears on retry. (2026-05-26)

- **A cached shim and the in-repo script can disagree mid-session** when the in-repo fix is still in the unreleased holding cluster; trust the in-repo script as ground truth and audit-trail-record the bypass per ADR-044 + ADR-013 Rule 5. (verifying P162; archived 2026-06-08)

## What Will Surprise You (archived 2026-06-10 — Tier 3 budget rotation, split-by-date)

- **`changeset publish` E404 on an EXISTING scoped package masks an npm-auth/2FA failure — NOT a missing package or transient blip.** When `'@windyroad/X@N' is not in this registry` fires while siblings "publish fine", the siblings only SKIPPED (already-published, read-only) — the new-version package is the only one exercising the publish token, and npm returns 404 instead of 403/EOTP to mask the auth failure. Diagnostic: `npm view X version` (confirms package exists) → `npm whoami` with the token (valid but no publish scope) → swap in a fresh token to surface `EOTP This operation requires a one-time password`. Root cause: a classic "Publish" token requires interactive OTP CI can't supply. Fix: regenerate as an **Automation / Bypass-2FA / Granular** token, update `NPM_TOKEN`/`NPM_AUTH_TOKEN` secrets, re-run `release:watch`. Until fixed, main self-poisons (Quality-Gates passes; only the publish job fails). See P284. Full ladder retained in Critical Points (`docs/briefing/README.md`). <!-- signal-score: 0 | last-classified: 2026-06-10 | first-written: 2026-05-25 -->
