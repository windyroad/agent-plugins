# Releases and CI

Cross-session learnings about push:watch, release:watch, changesets, npm publish, and the GitHub Actions pipeline.

## What You Need to Know

- **Atomic-cohort full graduation scales empirically to 13 changesets / 5 packages with no false-positives** (ADR-060 finding 12). Graduate a held cluster atomically (`git mv docs/changesets-holding/*.md .changeset/`) on user-comfort signal, not elapsed time. (verifying P162)
- **`.changeset/` empty + `docs/changesets-holding/` non-empty can self-reinforce a release-pressure stall**: ADR-042 auto-moves every changeset to holding, so the ADR-018/020 drain (needs `.changeset/` non-empty) never fires — commits accumulate unpushed AND nothing publishes. Only manual graduation recovers it. Monitor held-cluster size + last-push age proactively. (verifying P162)
- **Dogfood criterion is positive-evidence-of-working, NOT elapsed time.** Graduate a held entry when its surface fired correctly with no false-positive; KEEP HELD when it has negative evidence (e.g. hook misfired), regardless of days. Per-entry evidence beats sibling-cohort piggybacking. (verifying P246)
- **The `push:watch` script** in root `package.json` is the sanctioned way to push — it runs `git push` then watches the CI run. The git-push-gate hook blocks bare `git push`.
- **Never run `changeset version` locally.** The release pipeline handles versioning. To release: push with `npm run push:watch`, the pipeline creates a release PR, merge it with `npm run release:watch`.
- **Pre-2026-05-05 entries archived to [`releases-and-ci-archive.md`](./releases-and-ci-archive.md)** (rotated 2026-05-13 per Tier 3 MUST_SPLIT; further 2026-05-26 entries added 2026-06-08 — `push:watch false-green`, `.changeset/*.md external-comms gate`, `patchy network watch retry`). Load the archive when investigating older release / CI behaviour.
- **`.npmignore` alone is NOT load-bearing for `skills/*/eval/` tarball exclusion — the package.json `files:` `"!skills/*/eval/"` negation pattern IS.** When adding the first SKILL eval to a previously-eval-free plugin (Phase 4 of P324 hit this on `@windyroad/architect`), both files must be added: (a) `packages/<plugin>/.npmignore` with `skills/*/eval/`, AND (b) the negation pattern as a final entry in `package.json` `files:` array. npm honours `files:` negations BEFORE consulting `.npmignore`; missing the negation lets the eval files ship despite a correctly-patterned `.npmignore`. itil has both since RFC-012 S6; architect was missing the negation until P324 Phase 4 (2026-06-09). Diagnostic: `npm pack --dry-run | grep -cE "skills/.+/eval"` — zero matches required. ADR-075 §75 mechanism prose currently names `.npmignore` as the mechanism and under-describes this. <!-- signal-score: 2 | last-classified: 2026-06-09 | first-written: 2026-06-09 -->

## What Will Surprise You

- **Changing a SKILL invocation FORM breaks form-pinned STRUCTURAL contract tests in OTHER skills — only visible as red CI after push.** When you rewrite an invocation (e.g. `bash "$(... || echo packages/itil/scripts)/update-X-section.sh"` → the `wr-itil-update-X-section` shim, ADR-049), contract `.bats` that `grep -E 'update-X-section\.sh.*…'` go stale silently. RFC-009 cost TWO red→green CI round-trips this way (manage-story-map-contract, rfc-stories-extension). **Before pushing a form rewrite, `grep -rn '<old-form>' packages/*/skills/*/test/ packages/*/scripts/test/`** and update every form-pinned assertion. See P321. <!-- signal-score: 2 | last-classified: 2026-05-27 | first-written: 2026-05-27 -->
- **The full `bats --recursive` suite can HANG locally (no timeout) on `architect-detect-scope.bats`** — wedges with no error/progress for tens of minutes; the prior run completes, the re-run hangs (intermittent), and origin CI passes the same test. A hanging suite is worse than a failing one for the pre-push verify gate. Run TARGETED suites (the changed `.bats` files) for pre-push verify, not the full sweep; `pkill -f bats-core` to recover. See P319. <!-- signal-score: 2 | last-classified: 2026-05-27 | first-written: 2026-05-27 -->
- **`changeset publish` E404 on an EXISTING scoped package masks an npm-auth/2FA failure — NOT a missing package or transient blip.** When `'@windyroad/X@N' is not in this registry` fires while siblings "publish fine", the siblings only SKIPPED (already-published, read-only) — the new-version package is the only one exercising the publish token, and npm returns 404 instead of 403/EOTP to mask the auth failure. Diagnostic: `npm view X version` (confirms package exists) → `npm whoami` with the token (valid but no publish scope) → swap in a fresh token to surface `EOTP This operation requires a one-time password`. Root cause: a classic "Publish" token requires interactive OTP CI can't supply. Fix: regenerate as an **Automation / Bypass-2FA / Granular** token, update `NPM_TOKEN`/`NPM_AUTH_TOKEN` secrets, re-run `release:watch`. Until fixed, main self-poisons (Quality-Gates passes; only the publish job fails). See P284. <!-- signal-score: 0 | last-classified: 2026-05-25 | first-written: 2026-05-25 -->

- **macOS BSD awk vs Linux GNU awk produce different bytes from `awk substr(..., n) "…"` with Unicode ellipsis** — generators that use awk truncation produce machine-dependent output. `packages/architect/scripts/generate-decisions-compendium.sh` ships exactly this pattern (line 214 + emit_entry). Locally on macOS `--check` reports up-to-date; CI on Ubuntu fails the drift gate at test 2145 because GNU awk emits different bytes from identical ADR input. The compendium last regenerated locally on macOS won't match CI's GNU-awk regen → drift detected → release blocked. Fix paths: regenerate compendium on Linux (or with gawk locally) before pushing, OR replace awk-substr-with-Unicode-ellipsis with perl/python in the generator.
