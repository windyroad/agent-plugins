- **The `push:watch` script** in root `package.json` is the sanctioned way to push — it runs `git push` then watches the CI run. The git-push-gate hook blocks bare `git push`.
- **Never run `changeset version` locally.** The release pipeline handles versioning. To release: push with `npm run push:watch`, the pipeline creates a release PR, merge it with `npm run release:watch`.
- **Pre-2026-05-05 entries archived to [`releases-and-ci-archive.md`](./releases-and-ci-archive.md)** (rotated 2026-05-13 per Tier 3 MUST_SPLIT; further 2026-05-26 entries added 2026-06-08 — `push:watch false-green`, `.changeset/*.md external-comms gate`, `patchy network watch retry`). - **`.npmignore` alone is NOT load-bearing for `skills/*/eval/` tarball exclusion — the package.json `files:` `"!skills/*/eval/"` negation pattern IS.** When adding the first SKILL eval to a previously-eval-free plugin (Phase 4 of P324 hit this on `@windyroad/architect`), both files must be added: (a) `packages/<plugin>/.npmignore` with `skills/*/eval/`, AND (b) the negation pattern as a final entry in `package.json` `files:` array. npm honours `files:` negations BEFORE consulting `.npmignore`; missing the negation lets the eval files ship despite a correctly-patterned `.npmignore`. itil has both since RFC-012 S6; architect was missing the negation until P324 Phase 4 (2026-06-09). Diagnostic: `npm pack --dry-run | grep -cE "skills/.+/eval"` — zero matches required. ## What Will Surprise You

- **Commit author/commit DATES are non-monotonic after rebase — never decide "is this fix in release X" by comparing dates or by which `## <version>` heading the CHANGELOG entry sits under.** Witnessed P080 K→V iter 2026-06-15: the feat commit `5a4f8b1c` is dated 2026-06-09 but the `chore: version packages` commit that consumed its changeset (`43e164fe`) is dated 2026-06-08 — i.e. the changeset appears "deleted before it was added", and the CHANGELOG entry landed under `## 0.48.0` which sits BELOW `## 0.49.0` in the file. Both are rebase artefacts. **Ground truth = ancestry, not dates**: `git merge-base --is-ancestor <fix-commit> <latest-release-version-packages-commit>` → exit 0 means the fix IS in that release's published tree (npm ships the source tree at the version-packages commit). Cost ~3 tool calls reconciling the contradiction before falling back to ancestry. Composes with the existing `wr-itil-derive-release-vehicle` exit-2/exit-3 quirks (P361). <!-- signal-score: 2 | last-classified: 2026-06-15 | first-written: 2026-06-15 -->
- **macOS BSD awk vs Linux GNU awk produce different bytes from `awk substr(..., n) "…"` with Unicode ellipsis** — generators that use awk truncation produce machine-dependent output. `packages/architect/scripts/generate-decisions-compendium.sh` ships exactly this pattern (line 214 + emit_entry). Locally on macOS `--check` reports up-to-date; CI on Ubuntu fails the drift gate at test 2145 because GNU awk emits different bytes from identical ADR input. The compendium last regenerated locally on macOS won't match CI's GNU-awk regen → drift detected → release blocked. Fix paths: regenerate compendium on Linux (or with gawk locally) before pushing, OR replace awk-substr-with-Unicode-ellipsis with perl/python in the generator.
