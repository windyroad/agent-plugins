- **Dogfood criterion is positive-evidence-of-working-as-desired, NOT elapsed time** (P246, 2026-05-17 user direction). - **npm won't overwrite a published version**. Preview publishes use pre-release suffixes (e.g., `0.1.2-preview.13`) to avoid blocking `changeset publish` from publishing the clean version to `latest`.
- **The `push:watch` script** in root `package.json` is the sanctioned way to push — it runs `git push` then watches the CI run. The git-push-gate hook blocks bare `git push`.
- **Never run `changeset version` locally.** The release pipeline handles versioning. To release: push with `npm run push:watch`, the pipeline creates a release PR, merge it with `npm run release:watch`.

## What Will Surprise You

- **`changeset publish` E404 on an EXISTING scoped package masks an npm-auth/2FA failure — NOT a missing package or transient blip.** When `'@windyroad/X@N' is not in this registry` fires on `PUT .../@windyroad%2fX` while every sibling "publishes fine", the siblings only **SKIPPED** (already-published → read-only `npm info`, no token write); the new-version package is the ONLY one exercising the publish token. npm returns **404 instead of 403/EOTP** to mask the auth failure. Diagnostic ladder (3 round-trips wasted 2026-05-24 before landing it): (1) `npm view @windyroad/X version` confirms the package exists; (2) compare `publishConfig`/`private` to a publishing sibling (rules out packaging defect); (3) `npm whoami` with the token — if it returns the user, the token is *valid* but lacks publish scope; (4) **swap in a freshly-generated token → the real error surfaces as `EOTP This operation requires a one-time password`**. Root cause was the npm token lacked **"Bypass 2FA"** (a classic "Publish" token requires interactive OTP that CI can't supply; an **Automation** or **Granular** token bypasses it). Fix: regenerate as Automation/Bypass-2FA token, update the `NPM_TOKEN`/`NPM_AUTH_TOKEN` GitHub secrets, re-run `release:watch`. Until fixed, main is self-poisoning: every push retries the committed-but-unpublished version and re-fails the Release workflow (Quality-Gates passes; only the publish job fails). See P284. <!-- signal-score: 0 | last-classified: 2026-05-25 | first-written: 2026-05-25 -->
