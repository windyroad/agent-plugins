## What Will Surprise You

- **Copying an existing hook as a template silently propagates its latent BSD-awk portability bugs — and only the deny path exposes them.** The architect `architect-compendium-refresh-discipline.sh` (now retired by ADR-078 Story D) inlined a leading-`git commit` token check with `awk '/^git[[:space:]]+commit\b/'`. The `\b` word-boundary is a GNU-awk extension — BSD awk on macOS does NOT match it, so the check returns false, falls through `|| exit 0`, and the hook silently never fires its deny. Permit-path bats stay green either way (they expect exit 0); only a deny-path fixture caught it (P337 iter 10, 2026-06-16). Portable form: `commit([[:space:]]|$)`. Better still, the itil P165 sibling (`itil-readme-refresh-discipline.sh`) does NOT inline awk at all — it delegates to a shared, tested `lib/command-detect.sh::command_invokes_git_commit` helper (P268 family: tickets 268/272/274/275). Architect hooks should use the same shared helper rather than re-inlining commit detection. **Second instance (P392, 2026-06-27):** the blessed replace-section idiom in `update-problem-references-section.sh` (~L266) passes a multi-line section via `awk -v section="$multiline"` — BSD awk aborts with `awk: newline in string` while gawk tolerates it, so CI stays green and only a macOS dev hits it. Proven portable fix: write the section to a temp file and read it via `awk` `getline` (shipped in `effort-tally.sh --write`, commit 1c967ba0). Same template-copy-propagates-the-bug + inline-vs-shared-helper thesis. Same BSD/GNU awk-divergence class as P334/P328/P366/P392. **`find` sibling (P380, 2026-06-27):** `find <dir> -maxdepth 1` runs in `find`'s default `-P` (never-follow) mode; when `<dir>` is *itself* a symlink to a directory, `-P` won't descend it and emits nothing. On macOS the marker dir defaults to `/tmp` (a symlink to `/private/tmp`), so the `*-announced-*` candidate-SID enumeration in the oversight-marker shims + `session-id.sh::get_candidate_session_ids` silently returned zero and wrote zero markers (CI-Linux masks it — real `/tmp` there). Fix: `find -L <dir>` (follow symlinks; no-op on Linux). Subtlety that cost a wasted first test: `get_current_session_id` uses a *shell glob* (`ls .../*-announced-*`), which resolves through the symlink fine — so a behavioural test isolating the `find` bug needs a SECOND concurrent marker reachable only via the `find` path. <!-- signal-score: 4 | last-classified: 2026-06-27 | first-written: 2026-06-16 -->
